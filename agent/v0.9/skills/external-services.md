# 外部服务与后台任务

## 1. 加载条件

调用第三方 API、数据库、缓存、队列、对象存储、OCR、支付服务，或实现异步、定时和后台任务时读取。

## 2. 外部调用

必须考虑：

- 连接和读取超时
- 鉴权、限流、网络和服务不可用
- 返回状态、字段和数据格式校验
- 重试条件、次数、退避和幂等性
- 降级、补偿或人工恢复方式
- 调用耗时、结果摘要和追踪标识

不得默认第三方一定成功，也不得对所有失败盲目重试。

## 3. 后台任务

任务应具备：

- 明确状态和必要进度
- 超时、取消和失败记录
- 幂等和重复执行保护
- 受控重试与并发限制
- 用户可见的结果或错误
- 可定位的 task_id、trace_id 或等价标识

不能只把任务提交到后台而没有后续状态闭环。

## 4. 数据一致性

外部调用与本地写入组合时，说明：

- 调用与落库的顺序
- 任一步失败后的补偿
- 重复回调或重复请求处理
- 本地成功但远端失败，以及远端成功但本地失败的恢复方式

## 5. 验收

覆盖正常、超时、限流、鉴权失败、非法返回、服务不可用、重复执行和恢复流程。集成环境不可用时提供替代验证并明确未验证项。
