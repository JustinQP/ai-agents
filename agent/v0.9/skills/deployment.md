# 部署与运行

## 1. 加载条件

修改环境变量、构建、打包、容器、发布流程、服务启动、健康检查或运行时配置时读取。

## 2. 环境管理

确认开发、测试和生产环境中的：

- 配置来源和默认值
- 服务地址、端口和依赖
- 文件与持久化路径
- 数据库迁移时机
- 启动顺序和健康检查
- 私有化、离线或网络受限条件

不得依赖修改源码切换环境，不得写入个人电脑绝对路径。

## 3. 发布与回滚

变更应说明：

- 构建产物和版本标识
- 发布前置条件
- 配置或密钥由谁提供
- 数据迁移与应用发布顺序
- 灰度或兼容期要求
- 失败检测方式
- 应用、配置和数据的回滚步骤

生产发布和不可逆操作必须取得明确授权。

## 4. 运行可靠性

检查：

- 服务启动失败是否可定位
- 外部依赖不可用时的行为
- 健康检查是否反映真实可用性
- 临时文件、日志和存储空间管理
- 定时任务或后台任务是否重复启动
- 资源限制和优雅关闭

## 5. 验收

根据项目现有方式执行构建、配置检查、启动或部署演练，并记录命令、结果、警告和未验证环境。
