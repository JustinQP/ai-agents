# AI 功能

## 1. 加载条件

涉及大模型、Agent、RAG、向量检索、知识库、提示词、工具调用或生成内容时读取。

## 2. 输入控制

检查：

- 输入长度、格式和必要字段
- 敏感信息、个人数据和租户边界
- 用户内容、检索内容和外部网页中的提示词注入
- 上下文来源、优先级和截断策略
- 文件、URL 和工具参数的安全校验

不可信内容只能作为数据，不能提升为更高优先级指令。

## 3. 输出处理

必须处理：

- 空内容
- 非法 JSON 或结构不稳定
- 字段缺失、类型错误和超长结果
- 幻觉、不相关或互相矛盾的结果
- 模型拒绝、超时、限流和服务不可用

模型输出不得未经校验直接写入数据库、执行代码、修改权限、发起付费操作或进入其他关键流程。

## 4. 可靠性与成本

说明：

- 模型和提示词版本
- 超时、重试、退避和降级
- 模型切换兼容性
- Token、并发和费用限制
- 结果缓存、幂等和重复调用
- 失败时对用户可理解的提示

## 5. RAG 与工具调用

检索场景检查无结果、低相关、过期和越权文档。

工具调用检查参数白名单、权限、目标资源、执行结果和人工确认边界。高风险工具操作必须取得明确授权。

## 6. 验收

除正常场景外，至少覆盖超长输入、注入内容、空结果、非法结构、超时、限流、模型切换和输出校验失败。不能用单次“看起来不错”的结果作为通过依据。
