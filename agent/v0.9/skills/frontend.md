# 前端与交互

## 1. 加载条件

修改页面、组件、表单、列表、弹窗、路由、交互或前端状态时读取。

## 2. 状态覆盖

至少考虑：

- 首次加载和局部刷新
- 空数据
- 请求失败和重试
- 提交中和重复点击
- 权限不足和资源不存在
- 长任务等待、进度和完成反馈
- 乐观更新失败后的恢复

不能只实现有数据且请求成功的场景。

## 3. 表单与输入

- 前端校验用于及时反馈，不能替代后端校验。
- 错误提示应定位到具体字段或操作。
- 提交失败后保留用户可恢复的信息。
- 防止重复提交和意外多次触发。
- 涉及删除、覆盖和高风险操作时提供明确确认。

## 4. 兼容与可访问性

根据项目目标检查：

- 目标浏览器、终端和屏幕尺寸
- 键盘操作和焦点状态
- 语义化标签与必要的辅助说明
- 文案长度、空值和异常数据对布局的影响

不要擅自扩大为全面 UI 重构。

## 5. 验收

记录具体页面、前置条件、操作步骤、预期和实际结果。必要时验证加载、空态、失败、提交、权限和小屏幕场景。
