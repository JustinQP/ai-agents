# AGENTS.md

## 1. 仓库定位

本仓库是个人使用的 AI 指令与项目治理模板库，不是可运行的 Agent 框架。

维护目标：

- 沉淀长期稳定的个人全局指令
- 维护可复制到具体项目的 `AGENTS.md` 模板
- 将低频、专项规则拆分到独立文档
- 保证当前推荐版本可直接复制、路径有效、内容可验证

## 2. 当前有效版本

- 项目规则模板：`agent/v0.8/`
- Codex 全局指令：`codex-global/v0.1.md`
- ChatGPT 指令模板：`chatgpt-rule/v0.1.example.md`

`agent/v0.1` 至 `agent/v0.7` 作为历史版本保留。除非任务明确要求，不修改历史版本。

## 3. 修改规则

- 只修改与当前任务直接相关的文件。
- 修改当前项目规则时，优先更新 `agent/v0.8/`，不要复制出新的临时版本。
- 新版本必须是完整快照，不能只包含增量文件。
- `AGENTS.md` 中引用的相对路径必须在同一模板包内真实存在。
- 个人资料、账号、Token、密钥、公司敏感信息不得提交到仓库。
- 私人 ChatGPT 配置写入 `chatgpt-rule/local.md`，该文件由 `.gitignore` 排除。
- 文件和目录使用稳定、拼写正确的英文名称；正文默认使用中文。

## 4. 模板设计要求

项目模板必须区分：

1. 跨项目通用的行为约束
2. 使用者必须填写的项目特定信息
3. 按任务类型加载的专项规则

不得用通用模板替代项目的真实目录、命令、架构和验收标准。

## 5. 验证要求

修改后至少检查：

- `agent/v0.8/` 可整体复制到目标项目根目录
- `agent/v0.8/AGENTS.md` 引用的文件全部存在
- Markdown 使用 UTF-8、LF，并以换行结束
- README 中的当前版本和路径与仓库一致
- 不包含真实个人资料、密钥或本地绝对路径

未执行的验证不得声称通过。

## 6. 输出要求

完成修改后说明：

- 修改了什么
- 为什么修改
- 验证了什么
- 仍有哪些风险或未完成项
