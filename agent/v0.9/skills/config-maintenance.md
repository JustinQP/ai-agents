# AGENTS 配置维护

## 1. 加载条件

仅在修改 `AGENTS.md`、Skill、指令目录、当前模板版本或校验工具时读取。

## 2. 维护触发条件

出现以下情况时审查配置：

- Agent 第二次犯同类错误
- 项目命令、目录、架构或关键流程变化
- 代码审查发现 Agent 缺少必要上下文
- 同一纠正需要在多个会话重复说明
- 新增、合并或删除 Skill
- 距离上次审查超过维护元数据规定周期
- 发布新的模板版本

## 3. 六类配置异味检查

### Context Bloat

- 主 `AGENTS.md` 保持项目特定且简短。
- 总行数不得超过 `agents-config.json` 的限制。
- 示例、长流程和低频细节移入 Skill 或普通文档。

### Skill Leakage

- 仅高频、每次任务都需要的约束保留在主文件。
- 数据库、部署、安全、AI 等专项内容按触发条件放入独立 Skill。
- 不因方便而默认要求加载全部 Skill。

### Lint Leakage

- 格式、缩进、命名、导入顺序和行长等确定性规则交给格式化器、Linter、类型检查器或配置文件。
- 主文件只记录真实命令和工具无法表达的项目约束。

### Blind Reference

- 每个引用都必须说明触发条件、文件路径和用途。
- 引用路径必须存在，名称必须与内容职责一致。
- 不使用“详见某文件”但不说明何时读取的裸引用。

### Init Fossilization

- 删除过期命令、目录和流程。
- 更新 `agents-config.json` 中的版本和 `last_reviewed`。
- 更新版本记录，并运行校验工具。
- 不把初始化生成的内容永久当作事实。

### Conflicting Instructions

- 比较主文件、嵌套 `AGENTS.md` 和相关 Skill 的规范性语句。
- 删除重复但措辞不同的规则，避免范围不清。
- 保留明确优先级；同层冲突改写为单一规则。
- 安全、真实性和不可逆操作授权不得被低优先级规则放宽。

## 4. 修改流程

1. 确认修改解决了什么重复错误或项目变化。
2. 判断内容属于主文件、Skill、普通文档还是自动化工具。
3. 删除被替代或冲突的旧规则。
4. 更新维护元数据和版本记录。
5. 运行 `python tools/validate_agents.py`。
6. 记录仍需人工判断的冲突或风险。

不要仅增加新规则而不清理旧内容。
