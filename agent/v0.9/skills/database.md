# 数据库与数据变更

## 1. 加载条件

涉及表结构、字段、索引、约束、数据迁移、事务、状态流转或批量数据处理时读取。

## 2. 模型检查

确认：

- 主键、唯一约束和索引
- 字段类型、默认值和空值
- 状态枚举及合法流转
- 创建、更新时间和删除策略
- 与其他表中同义字段的一致性
- 现有查询、接口和报表的兼容性

不要使用易变化的业务展示字段作为稳定主键。

## 3. 迁移要求

迁移方案必须说明：

- 新环境如何创建
- 现有数据如何升级
- 大数据量下的执行时间和锁影响
- 默认值、空值和历史脏数据如何处理
- 失败后如何恢复
- 是否可以回滚
- 应在部署流程的哪个阶段执行

不可逆迁移或数据删除必须取得明确授权。

## 4. 一致性与事务

多表写入、多步骤状态变化、文件与数据库关联、计费和批量导入应使用事务、幂等设计或明确补偿机制。

无法强事务保证时说明：

- 可能失败的步骤
- 重试条件和次数
- 补偿动作
- 脏数据、孤立记录或残留文件的清理方式

## 5. 验收

至少验证：

- 全新数据库可完成迁移
- 代表性历史数据可正确升级
- 约束不会拒绝合法数据
- 重复执行不会产生错误结果
- 失败时不会留下半完成状态
- 回滚步骤真实可用
